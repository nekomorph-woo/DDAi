# 对话风格规范

## 1. 输出规范

### 1.1 变更摘要
- 格式：`📄 文件 → 📝 变更内容 → 📊 影响范围`

### 1.2 不确定性表达
- 明确标注：`需要确认...` / `建议验证...` / `置信度 X%`

### 1.3 进度反馈
- 多步骤任务：`■■■□□ 3/5` 或逐项 ✅

### 1.4 错误处理
- 格式：`❌ 原因 → 🔧 建议操作`

### 1.5 引用来源
- 标注位置：`file.py:42`
- 标注出处：`来源: 官方文档`

### 1.6 决策记录
- CLI 简述决策理由，不输出文件

## 2. 对话美学原则

### 2.1 可视化优先
- 流程/架构/系统交互 → 使用 ASCII 线框图
- **禁止** mermaid/plantuml 代码块（CLI 无法渲染）

### 2.2 方案先行
- 用户提问 → 先提供 1~4 个方案
- 方案内容：概要 + 关键细节 + 多方案对比
- **禁止**用户未决策前直接修改文件或开始执行代码开发，而是引导用户对计划文档或决策方案进行决策，得到用户同意后，再开始开发

### 2.3 工时估算
- 基于 AI Coding 快速编程，估算单位: **3m**

### 2.4 Emoji 增强
- 适当使用 emoji 增强语义表达和趣味性

### 2.5 关键信息优先
- 优先呈现核心结论/结果
- 细节按需展开

## 3. 安全规范

### 3.1 敏感信息检测
- **明文密钥检测**: Commit 前自动扫描以下模式
  - API Keys: `sk-`, `api_key`, `apikey`, `secret_key`
  - Tokens: `Bearer `, `token=`, `access_token`
  - Passwords: `password=`, `passwd=`, `pwd=`
  - Private Keys: `-----BEGIN.*PRIVATE KEY-----`
- **触发检测时**:
  - 🚨 立即停止当前操作
  - ⚠️ 向用户发出明确警告
  - 📄 标注疑似密钥位置（文件:行号）
  - 🔧 建议使用环境变量或密钥管理服务

### 3.2 安全响应格式
```
🚨 [SECURITY] 检测到疑似明文密钥
📄 位置: config.py:42
🔍 内容: api_key = "sk-xxxx..."
🔧 建议: 使用环境变量 API_KEY 或密钥管理服务
🚫 操作已中止，请确认后继续
```

## 4. 编码规范

### 4.1 兼容代码原则
- **默认不写兼容代码**: 除非用户明确要求，否则不在新代码中编写向后兼容逻辑
- **直接修改**: 直接替换旧实现，移除废弃代码
- **避免遗留**: 不保留 `_old`、`_legacy` 等命名标记
- **Rationale**: AI Coding 快速迭代，兼容代码增加维护成本且大多数情况下不必要

### 4.2 提交控制
- **禁止自动提交**: 完成编码任务后，**DO NOT** 自动执行 `git commit` 或 `git push`
- **等待用户指令**: 只有在用户明确发出 `commit`、`push` 或类似命令后才可执行
- **Rationale**: 用户可能需要审阅变更、调整提交内容或拆分提交，自动提交会剥夺这些控制权

### 4.3 脚本语言选择

当 Claude Code Agent 需要编写脚本处理任务时，按以下优先顺序选择语言：

| 语言 | 适用场景 | 约束 |
|------|----------|------|
| **Bash** | 文件/目录操作、简单命令组合 | 仅 Unix 系统 |
| **Python** | 数据处理、API调用、中等复杂逻辑 | 仅使用标准库，不引入三方依赖 |
| **TypeScript** | 复杂业务逻辑、与项目共享类型 | 必须提供 `npx ts-node` 运行方式或预编译 JS |

**优先级**：Bash > Python > TypeScript

**Python 标准库覆盖**：

| 功能 | 标准库 | 是否需要三方库 |
|------|--------|----------------|
| JSON/CSV/XML | `json`, `csv`, `xml.etree` | ❌ |
| HTTP 请求 | `urllib.request` | ⚠️ 可用但繁琐 |
| Excel 解析 | - | ✅ 需要 `openpyxl` |
| 文件系统 | `pathlib`, `os`, `shutil` | ❌ |

如需三方库，在 SKILL.md 中明确声明依赖。

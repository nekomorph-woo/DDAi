# Grill Me

对计划或设计进行系统性追问，逐层展开决策树并解决每个分支。

## 执行流程

1. **识别主题** - 确定要审视的计划或设计范围
2. **展开决策树** - 列出所有需要决策的分支点
3. **逐个追问** - 对每个分支提出具体问题：
   - 给出推荐方案
   - 说明理由
   - 记录用户选择
4. **代码库探索** - 若问题可通过代码库解答，优先探索代码库而非追问用户
5. **确认共识** - 回顾所有决策点，确认理解一致
6. **询问保存** - 达成所有共识后，询问用户是否持久化决策记录

## 追问原则

| 原则 | 说明 |
|------|------|
| **穷尽性** | 覆盖所有关键决策点，不遗漏分支 |
| **具体化** | 问题聚焦单一决策，避免笼统 |
| **提供方案** | 每个问题附带推荐答案及理由 |
| **先探后问** | 能从代码库获取的信息不追问用户 |

## 输出格式

```
## 决策点 1: <问题>
推荐: <推荐答案>
理由: <理由>
用户选择: [待确认]

## 决策点 2: <问题>
...
```

## 持久化

达成所有共识后，询问用户：

```
是否保存决策记录？[Y/n]
```

若用户确认，保存到：

```
<项目根目录>/grill-decisions/<YYYYMMDD>-<HHMM>-DECISIONS.md
```

格式详见 [reference/decision-format.md](reference/decision-format.md)。

## 示例场景

- 新功能实现方案审视
- 架构设计压力测试
- 技术选型决策梳理
- 重构计划可行性验证

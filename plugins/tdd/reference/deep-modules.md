# 深度模块

源自《软件设计哲学》。

**深度模块** = 小接口 + 大量实现

```
┌─────────────────────┐
│     小接口          │  ← 少量方法，简单参数
├─────────────────────┤
│                     │
│                     │
│    深度实现         │  ← 隐藏复杂逻辑
│                     │
│                     │
└─────────────────────┘
```

**浅模块** = 大接口 + 少量实现（避免）

```
┌─────────────────────────────────┐
│          大接口                 │  ← 大量方法，复杂参数
├─────────────────────────────────┤
│     薄实现                      │  ← 只是透传
└─────────────────────────────────┘
```

设计接口时，询问：

- 能否减少方法数量？
- 能否简化参数？
- 能否隐藏更多复杂度？

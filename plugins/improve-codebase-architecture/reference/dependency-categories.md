# 依赖类别

评估加深候选时，对其依赖进行分类：

## 1. 进程内（In-process）

纯计算、内存状态、无 I/O。始终可加深——直接合并模块并测试。

## 2. 本地可替换（Local-substitutable）

存在本地测试替身的依赖（如 PGLite 替代 Postgres、内存文件系统）。若测试替身存在则可加深。加深后的模块在测试套件中使用本地替身运行。

## 3. 远程但自有（Ports & Adapters）

跨网络边界的自有服务（微服务、内部 API）。在模块边界定义 port（接口）。Deep Module 拥有逻辑；传输层注入。测试使用内存适配器。生产环境使用真实 HTTP/gRPC/队列适配器。

推荐形式："定义共享接口，为生产环境实现 HTTP 适配器，为测试环境实现内存适配器，使跨边界逻辑可作为单个 Deep Module 测试。"

## 4. 真正外部（True external）

不可控的第三方服务（Stripe、Twilio 等）。在边界处 Mock。加深后的模块将外部依赖作为注入的 port，测试提供 Mock 实现。

# Issue 模板

<issue-template>

## 问题描述

描述架构摩擦：

- 哪些模块过浅且紧耦合
- 接缝之间存在什么集成风险
- 为何这让代码库更难导航和维护

## 提议接口

选定的接口设计：

- 接口签名（类型、方法、参数）
- 使用示例，展示调用者如何使用
- 内部隐藏的复杂度

## 依赖策略

依赖类别及处理方式：

- **进程内**：直接合并
- **本地可替换**：使用 [具体替身] 测试
- **Ports & Adapters**：port 定义、生产适配器、测试适配器
- **Mock**：外部服务的 Mock 边界

## 测试策略

- **新增边界测试**：描述在接口处验证的行为
- **需删除的旧测试**：列出变得冗余的浅模块测试
- **测试环境需求**：所需的本地替身或适配器

## 实现建议

持久架构指导：

- 模块应拥有的职责
- 模块应隐藏的实现细节
- 模块应暴露的接口契约
- 调用者如何迁移到新接口

</issue-template>

# PRD 转实现计划

使用垂直切片（tracer bullet）将 PRD 拆分为多阶段实现计划，输出至 `当前项目根目录/plans/`。

## 流程

### 1. 确认 PRD 在上下文中

PRD 应在当前对话中。若无，向用户询问 PRD 内容或文件路径。

### 2. 探索代码库

若尚未了解代码库，先探索以掌握当前架构、现有模式和集成层。

### 3. 识别持久架构决策

切片前，识别实现过程中不太可能变更的高层决策：

- 路由结构 / URL 模式
- 数据库 schema 形态
- 核心数据模型
- 认证 / 授权方式
- 第三方服务边界

这些决策放入计划头部，供所有阶段引用。

### 4. 起草垂直切片

将 PRD 拆分为 **tracer bullet** 阶段。每个阶段是贯穿所有集成层的薄垂直切片，而非单层的水平切片。

<vertical-slice-rules>
- 每个切片提供窄但完整的端到端路径（schema、API、UI、测试）
- 完成的切片可独立演示或验证
- 多个薄切片优于少数厚切片
- **DO NOT** 包含具体文件名、函数名或实现细节——这些在后续阶段可能变更
- **DO** 包含持久决策：路由路径、schema 形态、数据模型名称
</vertical-slice-rules>

### 5. 与用户确认

以编号列表展示拆分方案。每个阶段包含：

- **标题**：简短描述名称
- **覆盖用户故事**：对应 PRD 中的哪些用户故事

向用户确认：

- 粒度是否合适（过粗或过细）？
- 是否需要合并或进一步拆分？

迭代直至用户确认。

### 6. 编写计划文件

创建 `当前项目根目录/plans/` 目录（若不存在）。以功能名称命名 Markdown 文件，如 `plans/user-onboarding.md`。
